# Міні проєкт
## Варіант(завдання) №7
**Створити програму, яка реагує на сигнали SIGUSR1, SIGUSR2, SIGINT.
Наприклад, SIGUSR1 збільшує лічильник, SIGUSR2 виводить статистику, SIGINT завершує програму після підтвердження.
Бажано використати: sigaction(), sig_atomic_t, kill.**

Програма має запускатися в Linux з командного рядка. 
Код повинен містити перевірку помилок системних викликів. 
Потрібно підготувати короткий звіт: мета роботи, опис реалізації, використані системні виклики, приклад запуску, скріншоти роботи та висновок.

### Мета
Ознайомитися з механізмом обробки сигналів у Linux. 
Створити програму, яка безпечно та коректно реагує на користувацькі сигнали (`SIGUSR1`, `SIGUSR2`) та сигнал переривання (`SIGINT`), використовуючи сучасний стандарт `POSIX`.

## Опис реалізації
Замість застарілої функції `signal()` використано надійніший метод `sigaction()`.
У функції-обробнику `signal_catcher_supreme свідомо` не використовуються функції вводу-виводу (наприклад, `printf`), оскільки вони не є асинхронно-безпечними. Замість цього обробник лише змінює глобальні прапорці типу `sig_atomic_t`. Основний цикл програми знаходиться в стані очікування за допомогою `pause()`, і як тільки отримує сигнал - перевіряє прапорці та виконує відповідну логіку.

## Використані системні виклики та типи
* `sigaction()` - системний виклик для реєстрації обробника сигналів;
* `getpid()` - системний виклик для отримання ідентифікатора процесу (необхідний для надсилання команд через `kill`);
* `pause()` - системний виклик, що призупиняє виконання процесу до отримання будь-якого сигналу (економить ресурси CPU);
* `sig_atomic_t` - цілочисельний тип даних, який гарантує атомарність операцій зчитування/запису під час роботи із сигналами;
* утиліта `kill` - використовується з командного рядка для надсилання сигналів процесу.

### Код знаходиться в файлі `signal_tamer.c`
Щоб створити його у себе виконайте команду
```
nano signal_tamer.c
```

## Компіляція та запуск
Компіляція 
```
gcc signal_tamer.c -o signal_tamer
```

Запуск в першому терміналі
```
./signal_tamer
```
Програма виведе свій `PID`, і перейде в режим очікування

## Другий термінал 
Для взаємодії з програмою в другому терміналі треба виконати такі команди(для прикладу взято PID 1606)

Спочатку збільшимо лічильник втрийчі
```
kill -SIGUSR1 1606
kill -SIGUSR1 1606
kill -SIGUSR1 1606
```

Далі виведемо статистику
```
kill -SIGUSR2 1606
```

І в зрештою зробимо спробу завершити програму
```
kill -SIGINT 1606
```

## Висновок
Під час виконання завдання було успішно реалізовано обробку користувацьких сигналів у середовищі Ubuntu Linux. 
Програма коректно реагує на `SIGUSR1`, `SIGUSR2` та `SIGINT`, обробляє можливі помилки системних викликів. 
Застосовано правильний архітектурний підхід: використання `sigaction` для реєстрації та `sig_atomic_t` для безпечної передачі стану між обробником сигналів та основним потоком програми.
