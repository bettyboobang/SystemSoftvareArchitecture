**Практичне завдання №1**

Метою цієї практичної роботи було встановлення віртуальної машини та написання програми всередині вірт.машини. 
Через те, що віртуальну машину я встановлювала вперше, я стикнулася з деякою проблемою описаною нижче.

**Проблема.**
  При створенні віртуальної машини, і перших встановленнях убунту сервера, процес переривався або встановлювався некоректно.
  Після численних спроб, після встановлення серверу замість його налаштувань одразу з'являвся термінал і вимагав логін з паролем. Скоріше за все віртуальна машина не завантажилася з ISO-образу. Спочатку для вирішення цієї проблеми я спробувала скористатися методом з інтернету. Вимкнути ВМ, __зайти в налаштування -> storage -> controller: IDE(саме тут у файлу диска у мене було написано "Empty", я і подумала що це була проблема)-> вибрати файл з диску -> обрати ISO файл з убунту сервером._ Але це не змінило ситуацію і я продовжила шукати причину цієї проблеми. 

**Яке рішення пробеми я знайшла?**
Виявляється при створенні віртуальної машини, обираючи ISO файл для встановлення убунту сервера. Моя версія Oracle VirtualBox має позначку "Proceed with unattended installation" при якій _VM встановить ОС за допомогою автоматичної інсталяції під час створення віртуальної машини._

Тож щоб встановити налаштування Убунту сервера власноруч без ніяких проблем треба просто **прибрати галочку на цьому пункті**.

Ось так повиннен виглядати цей пункт для коректного встановлення серверу. Не у всіх виникають труднощі на цьому етапі але для мене це виявилось проблемою і я знайшла для неї рішення.

Після встановлення віртуальної машини і правильного налаштування Убунту сервера, треба написати програму вказану в завданні. Мій варіант 7, тож завдання у мене було таке: 

**Завдання 1.7**
_Напишіть програму на ANSI C, яка читає з stdin довільне ціле число — основу системи числення, потім ціле число у цій системі числення, а далі виводить його значення у десятковій системі. Ігноруйте всі недопустимі символи._

_Приклад:_

_Вхід:   8 77_   

_Вихід:  63 (значення 77 у системі числення з основою 8)_

_Додайте обробку значень із плаваючою точкою для десяткової системи числення._

Цю програму я назвала конвертором системи числення. Програма зчитує два значення: **базу системи числення(2-36)** і **число у цій системі(рядок)**

Після цього програма обчислює десяткове значення використовуючи алгоритм позиційного переведення. 

**Як це працює?**

Програма бере кожен символ введеного числа зліва направо. Якщо символ цифра, він перетворюється на число 0-9. Якщо символ буква, він перетворюється на число 10-35 (де A=10, B=11 ... Z=35). Коротко кажучи, я використала таблицю ASCII.

Для результату використовується формула: _новий результат = (старий результат * основа) + значення поточної цифри_

Отримане число виводиться на екран у звичній десятковій системі.


_**Чому я обрала число 36 як максимальне для системи числення?**_

База 36 є масимальною для стандартного запису оскільки маємо 10 цифр(від 0 до 9) і 26 латинських літер(від А до Z) -> **10 + 26 = 36**

**Як скомпілювати код, що знаходиться в файлі converter_lr1.c?**

Потрібно скомпілювати вихідний код за допомогою компілятора GCC ось цією командою:

_gcc -Wall converter_lr1.c -o converter_

Після чого запустити програму командою:

_./converter_
