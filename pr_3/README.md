## Завдання 1: 
**Запустіть Docker-контейнер і поекспериментуйте з максимальним лімітом ресурсів відкритих файлів.**
У Linux існують два типи лімітів для користувачів:
* М'який ліміт: Попереджувальний ліміт. Програма може його досягти, система відправляє сигнал, але не обов'язково одразу жорстко "вбиває" процес.
* Жорсткий ліміт: Абсолютна стеля. Звичайний користувач може зменшити свій жорсткий ліміт, але ніколи не може підняти його назад без прав адміністратора (root).
За замовчуванням Docker запускає контейнери від імені користувача `root`.
Оскільки перша частина вашого завдання передбачає роботу звичайного користувача, нам потрібно створити його.
Проте якщо до цього на віртальній машині не був встановлений Docker його треба встановити:
```
sudo apt update
```
```
sudo apt install docker.io -y
```
Після чого варто виконати наступні дії: 
1. Запускаємо інтерактивний контейнер з Ubuntu
```
docker run -it ubuntu bash
```
2. Створюємо звичайного користувача
```
useradd -m username
```
3. Перемикаємось на цього користувача
```
su - username
```

## Завдання 2
**У Docker-контейнері встановіть утиліту perf. Поекспериментуйте з досягненням процесом встановленого ліміту.**

Запустіть новий контейнер із прапорцем `--privileged`. Це дасть контейнеру дозвіл на використання `perf`:
```
sudo docker run -it --privileged ubuntu bash
```
Всередині контейнера оновіть списки пакетів та встановіть `perf`. У Linux ця утиліта постачається разом із пакетами `linux-tools`.
```
apt update
```
```
apt install -y linux-tools-common linux-tools-generic
```

В докері зазвичай немає редактору нано для подальшої роботи, тож спочатку встановимо його
```
apt install -y nano
```

Далі відкриймо файл для редагування
```
nano open_limit.c
```
Для подальшого експерименту використайте цей код:
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <errno.h>
#include <string.h>

int main() {
    int count = 0;
    int fd;

    printf("Start opening files...\n");

    while (1) {
        fd = open("/dev/null", O_RDONLY);
        
        if (fd == -1) {
            printf("\nStop on attempt %d!\n", count + 1);
            printf("Reason: %s (Error code: %d)\n", strerror(errno), errno);
            
            if (errno == EMFILE) {
                printf("Limit of open files is reached!\n");
            }
            break;
        }
        count++;
    }

    printf("In one time opened: %d files.\n", count);
    return 0;
}
```
Після чого скомпілюйте програму
```
gcc -Wall open_limit.c -o open_limit
```
Після цього можете сміливо запускати програму
Спочатку 
```
ulimit -n 100
```
А потім 
```
perf stat ./open_limit
```

## Завдання 6

**Напишіть програму, що демонструє використання обмеження (max stack segment size). Підказка: рекурсивна програма активно використовує стек.**
Використання нескінченної рекурсії - це найкращий спосіб показати, як вичерпується пам'ять стека і виникає славнозвісна помилка _Stack Overflow_

### Як це працює?
Коли функція викликає саму себе, системі потрібно запам'ятати її поточний стан (локальні змінні, параметри, куди повертатися після завершення). 
Всі ці дані складаються у спеціальну ділянку пам'яті - стек. 
Стек має жорстко обмежений розмір. Якщо рекурсія не має умови виходу, стек зростає, аж поки не впреться у свій ліміт. 
Тоді операційна система примусово завершує програму помилкою `Segmentation fault`.

**Щоб стек переповнився швидше, ми додамо всередину функції масив на 1 кілобайт**
Створіть файл програми, код програми знаходиться в файлі `stack_limit.c`
```
nano stack_limit.c
```
Далі скомпілюйте програму
```
gcc -Wall stack_limit.c -o stack
```
**Тепер перевіримо, як системний ліміт впливає на те, скільки разів функція зможе викликати сама себе.**
Зі стандартним лімітом
```
./limit
```

## Завдання 7(індивідуальне варіант 7)
**Використати ulimit -v для обмеження використання резидентної пам’яті.**
Резидентна пам'ять - це фізична пам'ять, яку дійсно займає програма, RSS. Параметр `ulimit -v` обмежує віртуальну пам'ять, доступну програмі. Якщо програма спробує попросити в операційної системи більше пам'яті, ніж дозволено цим лімітом, стандартна функція `malloc` просто поверне помилку (нульовий вказівник NULL). 

Спочатку знову створіть файл програми. Код знаходиться в файлі `memory_limit.c`
```
nano memory_limit.c
```
Скомпілюйте програму
```
gcc -Wall memory_limit.c -o memory
```
Тепер ми встановимо ліміт і подивимося, як програма з ним зіткнеться. 
Зверніть увагу, що для `ulimit -v` значення вказується у кілобайтах.
```
ulimit -v 102400
```
```
./memory_limit
```
