## Практична робота №6

### Завдання (Варіант 7)

**Написати програму, що активно використовує heap, потоки й mmap,
після чого проаналізувати її адресний простір через /proc/self/maps**

Програма повинна виконувати такі дії
* Виділення пам'яті на купі
Використовує функцію `malloc` для динамічного виділення пам'яті

* Створення потоків
Запускає додаткові потоки за допомогою бібліотеки `pthread`.
У Linux стеки для нових потоків створюються за допомогою системного виклику `mmap`.

* Пряме використання mmap
Використовує системний виклик `mmap` для виділення анонімної ділянки пам'яті великого розміру.

* Самоаналіз адресного простору
Програма читає та виводить у консоль вміст системного файлу `/proc/self/maps`, що дозволяє наочно побачити, де саме в пам'яті розташовані код програми, купа, `mmap`-ділянки та стеки потоків.

* Коректне звільнення ресурсів
Після завершення роботи програма звільняє всю виділену пам'ять (`free`, `munmap`) та коректно завершує потоки (`pthread_join`).

#### Створіть файл heap.c який знаходиться в іншому файлі цієї папки
```
nano heap.c
```

#### Компіляція та запуск
Для компіляції програми необхідно підключити бібліотеку `pthread` це стандартна бібліотека для створення та керування потоками всередині однієї програми

```
gcc -g -pthread heap.c -o heap
```

Ну а далі просто запустіть програму
```
./heap
```
